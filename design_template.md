## 0. 프로젝트 컨텍스트 (프롬프트 시작 부분에 그대로 붙여넣기)

나는 **천안시 청년 주거 안전 예측** 데이터 분석 대회에 제출할 발표 자료를 만들고 있어.
LightGBM 기반 깡통전세 위험 분류기(AUC 0.9893 / F1 0.9690)와 65개 법정동 안전 신호등 시스템이 핵심이야.
현재 만들어 둔 5개 figure는 정보는 다 들어있지만 **디자인이 밋밋하고 임팩트가 부족해서** 심사위원이 3초 안에 핵심을 못 잡아.
아래 지침대로 **matplotlib(+ 필요 시 plotly)** 코드를 재작성해줘. 결과물은 발표 슬라이드(16:9)에 그대로 붙일 수 있는 완성도여야 해.

## 1. 공통 디자인 시스템 (5개 figure 전부에 강제 적용)

### 1-1. 컬러 팔레트 (Semantic + Neutral)

프로젝트 특성상 **신호등 체계**가 이미 있으니, 이걸 전체 색 언어로 확장한다.

| 용도 | HEX | 사용처 |
|---|---|---|
| Safe (Green) | `#10B981` | 안전 등급, 긍정 지표 |
| Caution (Amber) | `#F59E0B` | 주의 등급, 중립 강조 |
| Risk (Red) | `#EF4444` | 위험 등급, 경고 |
| Ink 900 | `#0F172A` | 대제목, 큰 숫자 |
| Ink 600 | `#475569` | 본문 |
| Ink 400 | `#94A3B8` | 보조 설명, 축 라벨 |
| Line | `#E2E8F0` | 카드 테두리, 격자 |
| Surface Soft | `#F8FAFC` | 카드 배경 강조 |
| Risk Soft | `#FEF2F2` | 위험 강조 배경 |

**규칙**: 순수한 파랑(#1E88E5 등)은 쓰지 않는다. 지금 figure 1의 파랑-노랑-빨강은 신호등 3색으로 통일한다.

### 1-2. 타이포그래피

- **한글 폰트**: `Pretendard` (Variable). 없으면 `Noto Sans KR`.
- **숫자/영문**: `Pretendard` 또는 `Inter`. 폰트가 섞이지 않게 한 종류로 통일.
- **위계**:
  - `H1 / 큰 숫자`: 32–40pt, weight **700**, letter-spacing -0.02em
  - `H2 / 차트 제목`: 18–20pt, weight **600**
  - `Subtitle`: 13–14pt, weight 500, color `Ink 400`
  - `Body`: 12–13pt, weight 400
  - `Caption / Label`: 10–11pt, weight 500, letter-spacing 0.04em, **UPPERCASE 사용 지양** (한글이므로)

### 1-3. 레이아웃 & 스타일

- **8px grid**: 모든 여백은 8의 배수 (8/16/24/32).
- **카드**: `border-radius: 12px`, `border: 1px solid #E2E8F0`. 그림자는 쓰지 않는다 (또는 `0 1px 2px rgba(0,0,0,0.04)` 정도).
- **격자선**: 데이터 잉크 비율(Tufte) 원칙 — 축 격자는 y축 3~4단만 옅게(`#F1F5F9`), 나머지는 제거.
- **아이콘**: Lucide 또는 Phosphor의 얇은 선(1.5px) 스타일. 이모지 대체 금지.
- **색맹 대비**: 초록·빨강만으로 정보 구분 금지. 반드시 라벨/아이콘/명도차를 병기.

### 1-4. matplotlib 세팅 (모든 figure 최상단에 공통)

```python
import matplotlib.pyplot as plt
import matplotlib as mpl
from matplotlib import font_manager

# Pretendard 자동 등록
# (Pretendard-Variable.ttf 를 프로젝트에 넣거나 시스템 폰트 사용)
plt.rcParams.update({
    "font.family": "Pretendard",
    "axes.edgecolor": "#E2E8F0",
    "axes.linewidth": 0.8,
    "axes.labelcolor": "#475569",
    "axes.titlesize": 18,
    "axes.titleweight": "600",
    "axes.titlecolor": "#0F172A",
    "axes.spines.top": False,
    "axes.spines.right": False,
    "xtick.color": "#94A3B8",
    "ytick.color": "#94A3B8",
    "xtick.labelsize": 11,
    "ytick.labelsize": 11,
    "grid.color": "#F1F5F9",
    "grid.linewidth": 0.8,
    "figure.facecolor": "white",
    "axes.facecolor": "white",
    "savefig.dpi": 300,
    "savefig.bbox": "tight",
})

COLOR = {
    "safe":"#10B981", "caution":"#F59E0B", "risk":"#EF4444",
    "ink900":"#0F172A", "ink600":"#475569", "ink400":"#94A3B8",
    "line":"#E2E8F0", "soft":"#F8FAFC", "risk_soft":"#FEF2F2",
}
```

## 2. Figure별 재디자인 지침

### Figure 1 — 문제 통계 3-Numbers (`fig_Problem_Stats.png`)

**현재 문제**: 3개 카드가 크기만 크고 아이콘·서사가 없어서 "숫자만 나열"로 읽힘.

**개선**:
1. 카드 3개를 **인과 흐름**으로 배치: `19.7만명` → `86%` → `288세대` 사이에 얇은 회색 화살표(↘) 삽입.
2. 각 카드 좌상단에 **아이콘 뱃지** 추가 (사람 / 열쇠 / 경고 삼각형).
3. 색상은 신호등 체계 재사용: **초록 → 노랑 → 빨강** (현 상태 유지하되 HEX만 통일).
4. 큰 숫자 아래 `18~39세 청년 인구` 같은 라벨은 uppercase 대신 소문자 그대로, weight 500.
5. 각 카드 하단에 **얇은 progress bar** 하나 (예: 86% 카드는 86% 채워진 바). 시각적 지표 강화.
6. 하단 CTA "→ 도시 전체 선제 스캔 + 개인 매물 위험도 예측"은 **얇은 pill(둥근 알약) 박스**로 감싸고 `Ink 900` 볼드.

**전체 비율**: 16:6 정도 (가로 긴 히어로 섹션).

---

### Figure 2 — 모델 실험 히스토리 테이블 (`fig_Model_Compare.png`)

**현재 문제**: 순수 텍스트 표. 대회 심사위원이 어떤 실험이 왜 최고인지 즉시 못 봄.

**개선**:
1. AUC/F1 컬럼 옆에 **미니 수평 바 차트** 추가 — 각 실험 성능을 눈으로 비교 가능.
2. `exp_004` 행 전체를 `Surface Soft` 배경 + 좌측에 `Safe` 색 4px 액센트 바로 강조. `★`는 빼고 우측에 `BEST` pill 뱃지.
3. `AUC` 옆에 **Δ (기준 대비 증감)** 컬럼 추가: `+0.0172`처럼. 양수는 초록, 음수는 빨강 텍스트.
4. 컬럼 헤더는 검은 알약이 아니라 밑줄 있는 얇은 텍스트로. `AUC ↑`, `F1 ↑` 처럼 방향 아이콘.
5. 표 우측에 **iteration별 성능 line chart** (소형, 200×80px 정도) 추가 — 실험이 점진 개선됨을 시각화.
6. 각 행 사이 구분선은 `Line` 색 0.5px, 얼룩말 배경 제거.

**전체 비율**: 16:9.

---

### Figure 3 — 데이터 파이프라인 (`fig_Pipeline.png`)

**현재 문제**: 4개 컬러 박스가 나열되고 화살표가 약해서 "흐름"이 아니라 "리스트"로 읽힘.

**개선**:
1. 각 단계에 **큰 원형 번호 뱃지** (01, 02, 03, 04) + 아이콘.
2. 박스 색은 파랑/초록/보라/주황 대신 **모두 흰색 카드 + 상단 4px 액센트 바** (각 단계별 다른 톤: `Ink 600` 계열의 무채색 그라데이션으로).
3. 화살표는 얇은 실선 대신 **굵은 chevron(▶)** + gradient. 화살표에 짧은 설명 라벨 부착 가능 ("정제됨", "학습 완료" 등).
4. 각 박스 내부에 bullet 대신 **아이콘 + 텍스트 pair** 리스트 (예: 📄 실거래가·전세, 🏢 건축물대장).
5. 하단 `10만+ 실거래 · 65개 법정동 · 대회 규정 100% 준수`는 **얇은 stat strip**으로: 3개 metric을 얇은 divider로 구분해 배치.
6. 전체 카드에 얇은 `border 1px #E2E8F0` 통일.

**전체 비율**: 16:6.

### Figure 4 — 안전 신호등 도넛 (`fig_Grade_Donut.png`)

**현재 문제**: 도넛 하나만 덩그러니. Information density가 낮음.

**개선**:
1. 왼쪽 도넛 + 오른쪽 **등급별 대표 동네 top 3 리스트**를 side-by-side로.
   - 🟢 안전 17개 동: 불당동, 백석동, 청당동 …
   - 🟡 주의 47개 동: (대표 3개)
   - 🔴 위험 1개 동: 원성동
2. 도넛 중앙: `65` 위에 작은 라벨 `총 법정동`, 아래에 **`위험 1.5%`** 를 `Risk` 색으로 sub-metric 추가.
3. 도넛 세그먼트에 얇은 leader line + 수치 라벨 부착 (외부).
4. 도넛 굵기(width) 살짝 얇게 (0.35 → 0.28) — 모던한 인상.
5. 도넛 세그먼트 사이 2px 흰 gap 추가 (분리감).
6. 범례는 우측 리스트에 통합 (별도 legend block 제거).

**전체 비율**: 16:9.

### Figure 5 — 원성동 레이더 차트 (`fig_Radar_Wonseong.png`)

**현재 문제**: 축 라벨 작고, 어느 축이 얼마나 위험한지 즉시 파악 어려움.

**개선**:
1. 8개 축 라벨 크기 키우고(13pt, weight 600), 위험 축(값이 낮은 축: `건물노후`, `침수위험`, `금융안전`)은 **`Risk` 색 + 굵은 폰트**로 강조.
2. 각 축에 **평균 대비 Δ%** 를 라벨 아래 작게 표기: `건물노후 -34%`, `금융안전 -28%` 등.
3. 원성동 폴리곤을 `Risk` 색 + fill_alpha 0.15로, 평균 폴리곤을 `Ink 400` + 점선(dashed).
4. 축 tick 값 (0.25/0.5/0.75/1.0) 은 옅게(`Ink 400`), 3~4단만 표시.
5. 우상단 범례를 **badge 스타일**로: 색 점 + 텍스트, 배경 없이 깔끔하게.
6. 좌하단 또는 우하단에 **한 줄 요약 캡션** 추가:
   > "원성동은 건물 노후 · 금융안전 · 침수위험 3개 축에서 평균보다 30%+ 취약 → 즉시 개입 대상"

**전체 비율**: 1:1 (레이더는 정사각형 유지) 또는 4:3.

## 3. 결과물 요구 형식

1. **`viz_style.py`**: 위의 rcParams + COLOR dict + Pretendard 등록 함수를 담은 공통 모듈.
2. **`fig_01_problem_stats.py` ~ `fig_05_radar.py`**: figure별 독립 실행 가능한 스크립트. 각각 `outputs/` 폴더에 PNG 저장.
3. 각 파일 상단에 다음 주석 필수:
   ```python
   """Figure X — {제목}
   심사위원이 3초 안에 파악해야 할 메시지: {한 문장}
   """
   ```
4. 저장 옵션: `dpi=300`, `bbox_inches='tight'`, `facecolor='white'` (투명 배경 원할 시 인자로 토글 가능).
