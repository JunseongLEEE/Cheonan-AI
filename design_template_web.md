# 천안세이프 웹 디자인 템플릿 (Web Design System)
> 실서비스(app.py + styles/cinematic.py + scripts/listings.py) 코드에서 추출한 **현행 값** 기준.
> 발표 figure용 `design_template.md`(라이트/Pretendard/신호등)와 자매 문서 — 웹은 **시네마틱 다크** 테마.
> 8사이클 평가 루프(6.5→9.0)를 통과하며 확정된 규칙 포함. 2026-07-13 기준.

---

## 0. 디자인 철학 (평가 루프에서 검증된 3원칙)

1. **AI 가시성 철칙** — 화면의 모든 수치·색·순서는 모델 출력이며, 그 사실이 *보여야* 한다.
   (카드 배지 "위험확률 N% · 등급", 정렬 기본값 "AI 안전순", "앙상블 4모델 스코어" 캡션)
2. **정합성 철칙** — 같은 숫자는 어느 탭에서 봐도 같아야 하고, 다른 기준의 숫자는 기준을 명시한다.
   (원성동 44.9점 전 탭 일치 / "경보 19개 동(추세 기준)" vs "위험 1개 동(8축 기준)" 렌즈 캡션)
3. **오프라인 시연 철칙** — 외부 CDN(지도 타일 등)에 의존하는 시각 요소 금지. 지도는 자체 SVG 벡터.

## 0.5 천안 브랜드 아이덴티티

- **슬로건 체계**: 시 공식 "하늘 아래 가장 편안한 도시" → 히어로 타이틀 "하늘 아래에서, 계약 전 밤을 밝힙니다" / 서브 파생 "'하늘 아래 가장 편안한 도시' 천안의 청년…"
- **인증 라인** (히어로 하단 상시, 0.72rem `fg_subtle`):
  `천안시 공공데이터 · 국토부 실거래가 기반  |  본 서비스는 천안시청과 무관한 학술·시연 프로젝트입니다`
- **나랑이 마스코트** (`scripts/brand_assets.py` — base64 로컬 번들, 오프라인 철칙 준수):
  - 원본 배치 경로: `/root/Cheonan-AI/애국소녀 나랑이(2D, 이모티콘)/`, 심벌마크는 `/root/Cheonan-AI/심벌마크/`
  - 히어로 우측 170px + 블루 drop-shadow / AI상담 헤더 44px 원형(블루 보더) 아바타
  - 파일 부재 시 화면은 조용히 생략 (로더가 None 반환) — 포즈는 파일명 키워드로 선택(엄지/안내/놀람/응원)
- **핵심 카피 승격**: "동네와 매물은 따로 진단해야 합니다" — 히어로 서브카피 상시 노출

### 1-4. 신호등 4단계 (매물 카드 — scripts/listings.py)
빨강 일색 '양치기' 방지를 위해 위험 구간 세분화. **등급 = 천안 전체 단지 상대 분위**, %는 절대 위험확률 병기.

| 등급 | 분위 | 색 | 배경(라이트 카드) | 라벨 |
|---|---|---|---|---|
| 초록 | 하위 30% | `#10B981` | `#ECFDF5` | 안전 |
| 노랑 | 30~70% | `#F59E0B` | `#FFFBEB` | 주의 |
| 주황 | 70~90% | `#F97316` | `#FFF7ED` | 위험 |
| 빨강 | 상위 10% | `#DC2626` | `#FEF2F2` | 심각 |

- **규정 캡핑**: 전세가율 ≥80% 단지는 등급과 무관하게 '초록' 표기 금지 (최소 노랑)
- 위험확률 0%는 `<1%`로 표기 (과신 인상 방지)

### 1-5. 지도 그라데이션 (안전지도 choropleth)
안전점수 연속 보간: `40점 #EF4444 → 52점 #F97316 → 60점 #F59E0B → 72점 #10B981`
데이터 부족 폴리곤: `#4A5266` + 범례 "거래표본 부족" 필수.
**천안 실루엣**: 폴리곤 union 외곽선 `#5B8CE8` 1.6px + 블루 글로우 — "이건 천안 지도다"를 즉시 각인 (shapely, 실패 시 조용히 생략).

## 4. 핵심 컴포넌트 스펙

### 4-1. 매물 카드 (listings.card_html — 다크 위 라이트 카드)
```
┌──────────────────────────────┐  ← 좌측 4px 등급색 border-left (알림 관용 패턴)
│ [등급색 배경 헤더]                │ ← SIGNAL_BG, "● 위험확률 N% · 등급" + "앙상블 4모델 스코어"
│ [위험확률 게이지 6px]             │ ← 등급색, 폭=확률
│ 단지명(1.02rem/700 #0F172A)     │
│ 동 · 구 (0.86rem #475569)       │
│ 전세 N만원 (최근 실거래 YYYY.MM)  │
│ 면적 · 년식 · 거래 N건            │
│ ┄┄ 💡 근거 1줄 (전세가율 구간)     │ ← dashed 구분선
│ [네이버 매물검색↗][다방 지도(동네)↗][지도↗] │ ← pill 버튼, #F8FAFC bg
└──────────────────────────────┘
```
- 카드 내부는 **라이트 팔레트**(figure 시스템의 ink900/#0F172A, ink600/#475569, line/#E2E8F0) — 다크 배경과의 대비가 카드를 '떠 보이게' 함
- 그리드 3열, 카드 간 14px

### 4-2. 히어로 배너 (cinematic.hero)
아이브로우(모노, 보라) → 타이틀(gradient `<em>`) → 서브 → **pill 배지들**(청년 19.7만/경보 19개 동/AUC 0.9893/실거래 102,671) → 동선 힌트. 배경: 보라 radial glow + 24px 라운드 + 1px 보라 보더.

### 4-3. SVG 지도 2종 (오프라인)
- **안전지도**: 점수 그라데이션 채색, 위험동 `district-breathe` 맥동 애니메이션, 구도심 밀집 소형 동(원성1·2, 중앙, 문성, 일봉, 성정1·2)은 개별 라벨 생략 → **"원성동 일대(구도심) · 전세사기 288세대 피해 진앙" 단일 주석 + 점선 리더선**
- **추천 지도**: 비추천 동 `#252B38`, 추천 동만 등급색 + drop-shadow glow, 라벨 "🥇 병천리 / 70점" (흰 글자 + 검정 스트로크 3.5px, 인접 라벨 세로 엇갈림 −12px)

### 4-4. 라벨 가독 규칙 (다크 지도 공통)
`fill:#fff; paint-order:stroke; stroke:#000(0.85); stroke-width:3~4px` — 어떤 폴리곤 색 위에서도 판독 보장.

### 4-5. 원클릭 프리셋 칩 (매물 탐색)
🎓 대학가 5종(단국대·상명대·백석대/공주대/한기대/나사렛대/남서울대) + 🚄 KTX 천안아산역 + 🚇 1호선.
활성 시 "✓ " 접두 + 해제 안내 info. 위에 한 줄 캡션(칩 내부 줄바꿈 금지 — 사이클 7).

### 4-6. 외부 링크 (아웃링크 — 데이터 미수집)
| 버튼 | URL 패턴 | 검증 상태 |
|---|---|---|
| 네이버 매물검색 ↗ | `search.naver.com/search.naver?query=천안 {단지명} 전세 매물` | ✅ 실브라우저 (부동산 블록 최상단 착지) |
| 다방 지도(동네) ↗ | `dabangapp.com/map/onetwo?m_lat={}&m_lng={}&m_zoom=16` | 좌표 파라미터 유지 확인 (키워드 라우트 부재) |
| 지도 ↗ | `map.naver.com/p/search/천안 {동} {단지명}` | ✅ |

⚠️ 외부 개편으로 부패 가능 — 시연 전 재검증 필수. 하단 고지 문구 필수: "민간 플랫폼 데이터를 수집·저장하지 않습니다".

## 5. 카피라이팅 규칙

- 첫 화면은 항상 **AI 출력이 이미 있는 상태** (내매물체크: 원성동 사례 자동 진단 / AI상담: 실응답 로그 1턴 프리로드)
- 숫자엔 산출 기준 병기: "7,830건 (안전점수 산출용 전세 계약 기준)"
- 모델 내부어 금지: SHAP 축은 자연어("동네의 과거 위험 이력"), 0~1 점수는 /100 변환, 파일명·애니메이션명 노출 금지
- 킬러 카피: **"허위매물 0% — 실거래만 표시"** / "계약 전 3초" / "동네와 매물은 따로 진단"

## 6. 체크리스트 (새 화면 추가 시)

- [ ] AI 출력(확률·등급·순위)이 3초 안에 보이는가
- [ ] 이 화면의 숫자가 다른 탭과 충돌하지 않는가 (기준 다르면 캡션 명시)
- [ ] 외부 CDN 없이 렌더되는가
- [ ] 등급색만으로 정보 구분하지 않는가 (라벨 병기)
- [ ] 규정(전세가율 80%)과 화면 규칙이 모순되지 않는가
- [ ] 모델 내부어·원시 스케일이 새어 나오지 않는가
