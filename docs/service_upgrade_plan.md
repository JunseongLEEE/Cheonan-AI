# 천안세이프 v3 — "직방처럼 쓰고, AI가 지켜주는" 매물 탐색 플랫폼 플랜
> 목표: 발표장 시연에서 "이거 그냥 직방인데 **모든 매물에 AI 신호등이 붙어있네**"라는 반응을 만든다.
> 원칙: 화면에 보이는 모든 숫자·색·순서는 우리 AI 모델의 출력이어야 한다.

---

## 0. 포지셔닝 한 문장

> **"직방은 매물을 보여준다. 천안세이프는 매물의 '위험'을 보여준다."**

- 직방/다방과 경쟁이 아니라 **한 층 위의 안전 레이어**: 탐색 UX는 익숙하게(카드·지도·필터), 그 위에 AI 위험 진단을 상시 노출
- 결정적 차별점 카피: **"허위매물 0% — 우리는 실거래만 보여준다"**
  (매물 원천이 국토부 실거래 10.2만 건이므로 구조적으로 허위매물이 불가능 → 직방의 최대 고질병을 역공)

## 1. AI 사용 지점 맵 — "어디에 우리 모델이 쓰이나" (발표 강조 포인트)

시연 중 어떤 화면을 캡처해도 AI가 보이도록, 모델→화면 대응을 설계 단계에서 고정한다.

| 화면 요소 | 사용 모델 | 노출 방식 |
|---|---|---|
| 매물 카드의 신호등 배지·위험확률 | **exp_006 4-모델 앙상블** (LGBM+XGB+Cat+HistGB, 10.2만 건 전수 사전 스코어링) | 카드 좌상단 컬러 배지 + 게이지 |
| "안전순" 기본 정렬 | 앙상블 위험확률 + 8축 안전점수 합성 추천점수 | 정렬 드롭다운 기본값 |
| 매물 상세 "왜 위험한가" | **exp_004 LightGBM + SHAP** (실시간) | 워터폴 차트 + 한글 요인 3줄 |
| 지도 동네 색상 | 8축 안전점수 (65개 법정동) | choropleth + 매물 핀 신호등색 |
| 동네 헤더의 추세 화살표 | 시계열 분석 (월간 전세가율, 경보 19개 동) | 스파크라인 + "급상승 경보" 뱃지 |
| "이 단지 이상거래 이력" 경고 | **Isolation Forest** (5,134건) | 카드/상세의 보라색 경고 칩 |
| "이 매물 대신" 안전 대안 | 추천시스템 (동일 예산·면적대 재랭킹) | 상세 하단 대안 카드 3장 |
| HUG 보증 가능성 체크 | 규정 엔진 (공시가×126%·선순위 60%) | 상세의 체크리스트 위젯 |
| AI 상담 채팅 | **천안세이프-7B** (자체 파인튜닝, 4-툴 오케스트레이션) | 채팅 + 답변 안에 매물 카드 인라인 |

> 발표 멘트: "화면의 색 하나, 순서 하나까지 전부 모델 출력입니다. AI를 '탑재'한 게 아니라 AI로 '지은' 서비스입니다."

## 2. 핵심 화면 설계 (시각 임팩트 순)

### 2-1. 홈 = 검색 + 살아있는 신호등 지도
```
┌────────────────────────────────────────────────────┐
│  🔍 [ 동네·단지·예산으로 검색  "불당동 8000"      ] │
│                                                    │
│   ██ 천안 신호등 지도 (folium choropleth) ██        │
│   · 65개 동 안전점수 색칠 + 매물 핀(신호등색)        │
│   · 핀 호버: 단지명 · 보증금 · AI 위험 12%          │
│   · 우상단 토글: [시민 모드 | 시청 모드]            │
│  ── 하단 스트립: "오늘 AI가 스코어링한 매물 102,671" │
└────────────────────────────────────────────────────┘
```
- 임팩트 장치: 지도가 첫 화면 전체를 차지 (직방식). 시연 시작 3초 안에 "도시 전체가 진단돼 있다"가 보임
- "시청 모드" 토글 → 같은 지도가 B2G 대시보드(경보 19개 동·이상거래 히트맵)로 반전 — **한 번의 클릭으로 B2C→B2G 전환** (심사위원 순간 각인)

### 2-2. 매물 리스트 = 직방식 카드 그리드 + AI 배지 (시연의 심장)
```
┌─ 카드 ──────────────┐ ┌─────────────────┐
│ 🟢 AI 안전 7.4%     │ │ 🔴 AI 위험 89.0%  │  ← 앙상블 확률 (컬러 헤더)
│ ▓▓░░░░░░░░ 게이지    │ │ ▓▓▓▓▓▓▓▓▓░       │
│ 불당동 · OO아파트    │ │ 성정동 · XX빌라    │
│ 전세 9,000만 · 80㎡  │ │ 전세 8,000만 · 59㎡│
│ 2018년식            │ │ 2010년식 ⚡이상거래 │  ← IsolationForest 칩
│ "동네 위험이력 낮음,  │ │ "동네 평균 대비 높은 │  ← SHAP 요인 1줄 요약
│  신축 프리미엄"      │ │  보증금, 위험이력"   │
│ [실거래 이력] [지도]  │ │ [네이버부동산 매물↗] │  ← 아웃링크 (데이터 미수집)
│ [다방에서 보기 ↗]    │ │ [다방에서 보기 ↗]    │
└─────────────────────┘ └─────────────────┘
정렬: [AI 안전순 ▾] 필터: 예산 ▾ 면적 ▾ 신호등 ▾ 우선축(치안/교통) ▾
```
- Streamlit 한계 극복: `st.markdown(unsafe_allow_html)` 커스텀 CSS 카드 (design_template 팔레트 그대로 — 발표 figure와 앱이 같은 디자인 언어)
- 카드의 모든 요소가 §1 표의 모델 출력과 1:1 대응
- **아웃링크 규정 방어**: 네이버부동산·다방은 키워드 딥링크만 (단지명+동), 데이터 수집 0바이트. 카드 하단 고지: "매물 상세는 해당 플랫폼에서 · 본 서비스는 공공데이터만 저장"

### 2-3. 매물 상세 = "3초 판정 + 30초 설득"
```
┌──────────────────────────────────────────────┐
│  🔴 위험 89.0%   ← 풀폭 신호등 히어로 (애니메이션 게이지) │
│  성정동 XX빌라 · 전세 8,000만 · 59㎡ · 2010년식        │
├───────────────┬──────────────────────────────┤
│ SHAP 워터폴    │  동네 리포트 (성정동)           │
│ 위험이력 +0.86 │  · 8축 레이더 (vs 천안 평균)     │
│ 평균대비 +0.55 │  · 전세가율 12개월 스파크라인 📈  │
│ ㎡당가  -0.38  │  · 이상거래율 4.2% (평균 5.0%)   │
├───────────────┴──────────────────────────────┤
│ HUG 체크: 공시가×126% 초과 여부 ▢  선순위 60% ▢  │
│ 💡 이 예산이면 여기가 더 안전해요 (대안 3카드)      │
│ [🤖 AI상담에 물어보기] [🏛 안심계약 도움서비스 신청] │
└──────────────────────────────────────────────┘
```
- "이 매물 대신" 대안 3장이 킬러: 위험 매물을 보던 사용자를 **모델이 안전 매물로 손잡고 이동**시킴 — 추천시스템의 실효성이 행동으로 증명됨

### 2-4. AI 상담 = 채팅 안에 매물 카드가 뜬다
- 천안세이프-7B의 `recommend` 툴 결과를 텍스트가 아니라 **§2-2 카드 컴포넌트로 인라인 렌더**
- 시연 대사: "예산 7천으로 교통 좋은 데 추천해줘" → 챗봇이 말로 설명 + 카드 3장 등장 + 각 카드에 아웃링크
- 이미 검증된 멀티턴("거기서 8천이면?")을 카드와 함께 보여주면 데모 정점

## 3. 시연 시나리오 (발표장 3분, 실패 지점 없음)

| 시간 | 행동 | 심사위원이 보는 것 |
|---|---|---|
| 0:00 | 홈 지도 오픈 | 도시 전체 신호등 — "선제 스캔" 즉시 각인 |
| 0:20 | "성정동 8000" 검색 → 리스트 | 카드마다 AI 확률·SHAP 요약·이상거래 칩 |
| 0:50 | 🔴 카드 클릭 → 상세 | 게이지 애니메이션 → 89% + 워터폴 "왜" |
| 1:20 | "이 매물 대신" 대안 클릭 | 모델이 안전 대안으로 유도 (실효성) |
| 1:40 | [다방에서 보기↗] 클릭 | 실제 플랫폼 연결 — "오늘부터 쓸 수 있음" |
| 2:00 | AI 상담: "거기 말고 치안 좋은 데는?" | 7B가 툴 호출 → 카드 3장 인라인 |
| 2:40 | 시청 모드 토글 | B2G 경보 대시보드 — 정책 활용 마무리 |

백업: 전 구간 녹화 영상 + 노트북 로컬(GGUF) 이중화 (기존 계획 유지)

## 4. 구현 우선순위 (코딩 착수 시)

**P0 — 시연 필수 (2~3일)**
1. `recommender.py`에 아웃링크 필드 (네이버부동산·다방 키워드 딥링크 + 국토부 실거래) — 30분
2. 매물 카드 컴포넌트 (커스텀 CSS, 신호등 헤더 + 게이지 + SHAP 1줄 + 칩 + 링크) — 핵심 공수
3. 매물 리스트 화면 (검색 → recommend_base 필터 → 카드 그리드, AI 안전순 정렬)
4. 매물 상세 (신호등 히어로 + SHAP 워터폴(plotly) + 대안 3카드 + HUG 체크)

**P1 — 임팩트 증폭 (1~2일)**
5. 홈 지도 개편 (choropleth + 매물 핀, folium) + "오늘 스코어링 N건" 스트립
6. 챗봇 답변 내 카드 인라인 렌더
7. 시민/시청 모드 토글 (기존 B2G 요소 재배치)

**P2 — 완성도**
8. 동네 스파크라인·이상거래 칩 전면 적용, 게이지 애니메이션, 스켈레톤 로딩
9. 시연 영상 녹화 + GGUF 노트북 이중화

## 5. 리스크와 방어

| 리스크 | 방어 |
|---|---|
| "직방 데이터 쓰는 거 아니냐" (규정) | 아웃링크만·수집 0바이트 — 화면과 기획서에 명시 고지 |
| 직방 딥링크 구조 변경 | 네이버부동산·다방 우선 (URL 안정), 시연 전주 링크 점검 스크립트 |
| "실거래 = 지금 매물 아님" 지적 | 명칭을 "실거래 기반 매물 프로필"로 정직하게 + "현재 매물은 링크에서" — 오히려 허위매물 0% 서사로 전환 |
| Streamlit 카드 UI 한계 | 커스텀 HTML/CSS (figure와 동일 팔레트) — 이미 검증된 접근 |
| 발표장 인터넷 | 지도 타일 오프라인 캐시 + GGUF 로컬 LLM + 녹화 백업 |
