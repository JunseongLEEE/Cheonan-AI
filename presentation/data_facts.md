# 데이터 사실 노트 (data_facts.md)

> **용도**: Claude Design(PPT 자동화) + 사람이 발표 스크립트를 검증할 때 쓰는 "숫자 원본".
> 여기 있는 수치·인용은 모두 프로젝트 산출물(파케이/CSV/JSON/실험 로그)에서 직접 도출한 값이다.
> 슬라이드에서 숫자를 인용할 때는 반드시 이 문서에서 복사한다.

## 1. 문제의 크기 (Why now)

| 지표 | 값 | 출처 |
|---|---|---|
| 천안시 18~39세 청년 인구 | **197,572명** | `CLAUDE.md · YOUTH_POPULATION` (SGIS 2025.5) |
| 천안시 총인구 | 661,615명 | 동일 |
| 청년 주택 자가보유율 | **13.9%** (=무주택 **86.1%**) | 동일 · `YOUTH_HOMEOWNERSHIP` |
| 천안시 전세사기 피해 (2024) | **288세대 / 145억 원** | `CLAUDE.md · FRAUD_HOUSEHOLDS / FRAUD_AMOUNT_BILLION` |
| 피해 집중 지역 | 원성동 (동남구 구도심) | 안전점수 최하 동네와 일치 (아래 §4) |

> 핵심 메시지: 청년 인구 20만, 그중 86%가 세입자. 그런데도 **자취방 계약 전에 "이 동네·이 매물"이 안전한지 진단해 주는 통합 서비스가 없다.**

## 2. 데이터 파이프라인 (What we ingested)

**대회 규정 준수**: 공공 API + 공공데이터포털만 사용. 민간(직방/다방/네이버부동산) 크롤링 **0건**.

|---|---|---|---|
| 실거래 (매매) | 국토부 실거래가 API | 다년치 | 거래금액, 전용면적, 건축년도 |

### 2-2. 처리 산출물 (`data/processed/`)

## 3. AI 모델 (What we built)

### 3-1. 깡통전세 분류기 (exp_004_threshold_80) — 대표 모델

| 항목 | 값 |
|---|---|
| 알고리즘 | LightGBM (binary) |
| 라벨 정의 | 전세가율 ≥ 0.80 = 위험, ≤ 0.60 = 안전, 60~80 % 미분류 |
| 학습 샘플 | **60,412건** (positive 46,084 / negative 14,328) |
| CV | 5-fold stratified, seed 42 |
| **AUC** | **0.9893** (± 0.00106) |
| **F1** | **0.9690** |
| 설명력 | SHAP TreeExplainer (전체 학습셋) |

### 3-2. SHAP 상위 5개 피처 (mean |SHAP|)

`experiments/exp_004_threshold_80/shap/shap_importance.csv`

1. **연도별_동_위험도** — 2.096
2. **건물연령** — 1.451
3. **㎡당_보증금** — 1.020
4. **거래연도** — 0.763
5. **건물연령_동평균차** — 0.470

> 스토리라인: "모델이 본능적으로 짚어낸 위험 신호는 (1) 이 동네가 최근 위험해지고 있는가, (2) 건물이 얼마나 낡았는가, (3) 이 건물이 동네 평균 대비 비싼가 — 세 가지."

### 3-3. 실험 히스토리 (LEADERBOARD)

| ID | 실험 | AUC | F1 | 핵심 변경점 |
|---|---|---|---|---|

| exp_002 | no_trade_price | 0.9683 | 0.9385 | 실거래가 컬럼 제외 (데이터 누수 제거) |

### 3-4. 이상탐지 (Isolation Forest)

- 전체 전세 실거래 **102,671건** 스캔
- 이상거래 탐지 **5,134건** (전체 5.0%)
- 고전세가율(≥0.90) 매물 **19,310건**

### 3-5. 8축 안전점수 (동네 단위 통합)

**축**: 금융안전 · 건물노후 · 침수위험 · 치안 · 소방 · 교통 · 편의시설 · 환경
**결과**: 종합점수(0~100) + 신호등(🟢/🟡/🔴) 65개 동에 할당

## 6. 인용 규칙 (Slide에서 이 숫자를 쓸 때)

- 숫자는 이 문서에서 **그대로 복사**할 것 (반올림/의역 금지).
- 항상 "**공공 API 기반**" 또는 "**공공데이터포털**"이라는 라벨을 붙일 것 (심사위원이 데이터 규정을 재확인함).
- exp_004는 **정식 명칭 "exp_004_threshold_80"** 로 표기 (README/실험 로그와 일치).
- "AUC 0.9893 / F1 0.9690" — 두 값은 항상 세트로 인용.
- 신호등 개수(1/47/17)는 순서를 (빨강→노랑→초록)으로 통일.
