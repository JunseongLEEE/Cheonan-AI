## Symptom
대화 프로브(18개 시나리오)에서 발견된 문제:
오타 동네를 기본 통계로 오진("불당둥"→안전 3.8%), HUG 조건 창작("소득 36배"),
중국어/태국어 혼입, 프롬프트 인젝션에 가짜 안전점수 생성(원성동 88→95/100, 실제 44.9),
후속 턴에서 이전 진단 수치 변조(15.57%→35.21%).

## Root cause
- 시뮬레이터가 미등록 동을 조용히 기본값으로 fallback (툴 쪽 검증 부재)
- 파인튜닝 지식만으로 정책 세부사항 답변 (근거 주입 없음)
- temperature 0.2 샘플링에서 저확률 다국어 토큰 유출
- 7B는 시스템 프롬프트의 금지 규칙만으로는 인젝션·수치 창작을 막지 못함

## Fix (전부 서빙단, 재학습 불필요)
1. 툴 실행기에서 법정동 검증: difflib fuzzy 교정('불당둥'→'불당동'), 읍/면은 하위 리 후보 반환
2. KB 문자-bigram 검색으로 정책 문서를 시스템 프롬프트에 주입 ('보증보험'↔'반환보증' 표기차 대응)
3. temperature 0 (greedy) → 언어 혼입 소멸
4. 그라운딩 가드 1: 동네 언급 + 툴 미호출 + 수치 주장 → dong_lookup 강제 실행 후 재작성
5. 그라운딩 가드 2: 툴 없는 턴의 지표 수치가 대화 이력에 없으면 1회 재작성 요구
6. 규칙 정밀화 교훈: "동네 언급 시 무조건 툴 호출" 같은 광범위 지시는 부작용
   (서울 동네에 반응, 동네 지어내서 호출) — 조건을 좁게 명시해야 함

## Generalization
sLLM 에이전트의 신뢰성은 모델 능력이 아니라 **툴 입력 검증 + 근거 주입 + 결정적 사후 검증**의
3중 가드에서 나온다. 가드는 재학습 없이 배포 가능하므로 발견→수정 사이클이 분 단위로 돈다.
